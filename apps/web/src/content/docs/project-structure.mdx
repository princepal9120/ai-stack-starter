## Overview

AI Stack CLI scaffolds a production-ready project with organized directories for frontend, backend, and AI components. The exact structure depends on your architecture choice.

## Root Layout

At the repository root you will see:

```
my-ai-app/
├── app/                    # Next.js App Router
├── components/             # React components
├── lib/                    # Shared utilities
├── types/                  # TypeScript types
├── public/                 # Static assets
├── .env.example           # Environment template
├── package.json
├── tsconfig.json
├── next.config.ts
└── README.md
```

## Key Directories

### `lib/ai/`

LLM provider abstraction:

```
lib/ai/
├── index.ts               # Main exports
├── config.ts              # Provider configuration
├── providers/
│   ├── openai.ts
│   ├── anthropic.ts
│   ├── novita.ts
│   └── ollama.ts
└── types.ts
```

### `lib/db/`

Database configuration:

```
lib/db/
├── index.ts               # Database client
├── schema/                # Drizzle or Prisma schema
│   ├── users.ts
│   ├── conversations.ts
│   └── documents.ts
└── migrations/            # Migration files
```

### `lib/search/`

Vector search abstraction:

```
lib/search/
├── index.ts               # Main exports
├── config.ts
├── providers/
│   ├── qdrant.ts
│   ├── weaviate.ts
│   └── pgvector.ts
└── types.ts
```

### `lib/auth/`

Authentication setup:

```
lib/auth/
├── index.ts
├── config.ts              # Auth provider config
└── middleware.ts          # Auth middleware
```

## Configuration Files

### `next.config.ts`

```typescript
import type { NextConfig } from 'next';

const config: NextConfig = {
  experimental: {
    typedRoutes: true,
  },
};

export default config;
```

### `drizzle.config.ts`

```typescript
import { defineConfig } from 'drizzle-kit';

export default defineConfig({
  schema: './lib/db/schema',
  out: './lib/db/migrations',
  dialect: 'postgresql',
  dbCredentials: {
    url: process.env.DATABASE_URL!,
  },
});
```

## Key Details

- **Type-safe**: Full TypeScript across the stack
- **Modular**: Each provider is a separate module
- **Swappable**: Change providers via configuration
- **Production-ready**: Includes error handling, logging
